# GPS HUD – ESP32 + ATGM336H

A real-time GPS Head-Up Display (HUD) system built for ESP32 platforms (ESP-IDF 5.x/6.x) with the ATGM336H GPS module and 3.5-inch LCD touchscreen.

## Overview

This project implements a complete GPS data processing pipeline that reads NMEA sentences from an ATGM336H module, parses GGA and RMC messages, synchronizes the ESP32 system RTC, and displays real-time navigation data on an LVGL-powered touchscreen interface.

## Features

### Core Functionality
- **NMEA 0183 Parser** – FSM-based parser for GGA and RMC sentences with checksum validation
- **Lock-free Double Buffer** – Thread-safe GPS data sharing between tasks without mutex overhead
- **Auto Baudrate Detection** – Automatically detects and configures GPS module to 115200 baud
- **RTC Synchronization** – Syncs ESP32 system clock with GPS UTC time, displays GMT+7 (Vietnam time)
- **Odometer & Speed Filtering** – Exponential Moving Average (EMA) speed filter with distance tracking
- **Signal Quality Indicator** – HDOP-based signal strength visualization

### Display Features
- Real-time speed display (km/h) with user-adjustable compensation (+0 to +4 km/h)
- Position display (latitude/longitude in DMS format)
- Satellite count and HDOP information
- Moving status indicator
- Signal strength icon (Excellent → Good → Moderate → Bad → No Signal)
- RTC time/date with weekday (automatically updates every second)
- Odometer tracking (meters/km)
- NMEA sentence rate statistics (for debugging)

### Hardware Support
- CYD 3.5 inch with following:
- ST7796 3.5-inch LCD (320x480) via SPI
- XPT2046 resistive touch controller
- RGB LED indicators (Red: error, Green: RTC sync, Blue: unused)
- Configurable UART GPS interface

## Data Flow Architecture

```
UART (GPS)
    ↓
NMEA FSM Parser (gps_nmea.c)
    ↓
Double Buffer (lock-free swap)
    ↓
Task Notifications
    ├── gps_task → rtc_task (EVT_RTC_SYNC_REQUEST)
    ├── gps_task → ui_task (EVT_GPS_UPDATE, EVT_GPS_TIMEOUT, etc.)
    ├── rtc_task → gps_task (EVT_RTC_SYNC_DONE)
    └── rtc_task → ui_task (EVT_RTC_SYNC_DONE - flash icon)
```

## Tasks & Core Affinity

| Task | Core | Priority | Stack Size | Function |
|------|------|----------|------------|----------|
| `lvgl_port_task` | 1 | 6 | 8KB | LVGL rendering, calls lv_timer_handler() |
| `ui_task` | 1 | 5 | 6KB | UI updates, event handling |
| `gps_task` | 0 | 6 | 3KB | UART reading, NMEA parsing, odometer |
| `rtc_sync_task` | 0 | 4 | 2KB | Executes RTC sync when requested |
| `debug_task` | 1 | 2 | 2KB | Optional serial debug output |

*Tasks on multi-core chips (ESP32/ESP32-S3) are pinned to separate cores for optimal performance.*

## NMEA Messages Processed

The parser extracts data from two sentence types:

### GNGGA ($GNGGA)
- Latitude / Longitude
- Altitude (meters)
- HDOP (Horizontal Dilution of Precision)
- Satellite count

### GNRMC ($GNRMC)
- UTC Time (HHMMSS.sss)
- Fix status (A=valid, V=void)
- Latitude / Longitude
- Speed over ground (converted to km/h)
- Course over ground (degrees)
- Date (DDMMYY)

## GPS Module Configuration

The system automatically configures the ATGM336H module with these commands:

| Command | Description |
|---------|-------------|
| `$PCAS01,5*19` | Set baudrate to 115200 |
| `$PCAS02,100*1E` | Set update rate to 10Hz |
| `$PCAS03,1,0,0,0,1...` | Enable GGA and RMC only |
| `$PCAS04,13,13,11*29` | Enable GPS + BeiDou constellations |

*Auto-baudrate detection probes: 115200, 9600, 38400, 57600, 230400, 460800*

## Key Algorithms

### NMEA Field Parsing
- No `strtok()` (non-thread-safe)
- No `sscanf()`/`atof()` (reduced overhead)
- Manual integer and floating-point parsers with direct pointer manipulation

### Lock-free Double Buffer
```c
// Writer (gps_task)
uint8_t next = g_gps.index ^ 1;
g_gps.buf[next] = new_data;
__atomic_store_n(&g_gps.index, next, __ATOMIC_RELEASE);

// Reader (ui_task, rtc_task)
uint8_t idx1, idx2;
do {
    idx1 = __atomic_load_n(&g_gps.index, __ATOMIC_ACQUIRE);
    *out = g_gps.buf[idx1];
    idx2 = __atomic_load_n(&g_gps.index, __ATOMIC_ACQUIRE);
} while (idx1 != idx2);
```

### EMA Speed Filter & Odometer
```c
// Exponential Moving Average
g_ema_speed_kmh = alpha * current_speed + (1-alpha) * g_ema_speed_kmh;

// Distance accumulation
if (g_ema_speed_kmh > MIN_ODO_SPEED) {
    g_odometer_m += (g_ema_speed_kmh / 3.6) * delta_t_sec;
}
```

## Event System

FreeRTOS task notifications with bitmask flags:

| Event | Sender | Receiver(s) | Purpose |
|-------|--------|-------------|---------|
| `EVT_GPS_UPDATE` | gps_task | ui_task, debug | New GPS data available |
| `EVT_RTC_SYNC_REQUEST` | gps_task | rtc_task | Request RTC synchronization |
| `EVT_RTC_SYNC_DONE` | rtc_task | gps_task, ui_task | Sync completed |
| `EVT_GPS_TIMEOUT` | gps_task | ui_task | No data for 2+ seconds |
| `EVT_GPS_RESTORED` | gps_task | ui_task | GPS recovered from timeout |
| `EVT_RTC_STALE` | gps_task | ui_task | RTC not synced for 2+ hours |
| `EVT_RTC_NOT_STALE` | gps_task | ui_task | RTC freshly synced |
| `EVT_VALID_CHANGE_TO_*` | gps_task | ui_task | Fix status changed |
| `EVT_IS_MOVING_TO_*` | gps_task | ui_task | Motion status changed |
| `EVT_SIGNAL_CHANGE` | gps_task | ui_task | Signal level changed |
| `EVT_GPS_STATS_UPDATE` | gps_task | ui_task | New NMEA rate stats |

## Troubleshooting

### "Auto baudrate detection failed"
- Check GPS module connections (RXD must be connected)
- Verify GPS module is powered and has antenna connected
- Check if GPS module outputs any NMEA sentences via serial monitor

### "baudrate err!" - Red LED blinking continuously
- GPS module not responding at detected baudrate
- Check UART wiring and power supply
- Try manual baudrate configuration

### No satellite fix (display shows "NO GPS" or "NOT FIXED")
- Ensure GPS antenna has clear sky view
- Wait 30-60 seconds for cold start acquisition
- Check HDOP value - higher values indicate poor signal

### RTC not syncing
- GPS must have valid fix (`d->valid == true`)
- Check UTC time is present in GNRMC sentence
- Verify `gps_rtc_should_sync()` conditions

### LVGL rendering issues / watchdog timeouts
- Increase `LVGL_TASK_MIN_DELAY_MS` if CPU is overloaded
- Reduce draw buffer lines if memory is constrained
- Disable ESP Task Watchdog in menuconfig during development
